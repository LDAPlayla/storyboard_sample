기능 데모 토글
유저스토리 US-01 ~ 12 ▲
US-01 좌석 수 기준 렌더 분기 ▼
공연 규모에 맞는 도면을 즉시 본다.
소규모(≤3,000석)는 바로 좌석을, 대규모는 구역 조감도를 먼저 표시.
✓ 구현됨 (Case A/C vs B/D)
US-02 구역 조감도에서 빈자리 파악 ▼
조감도에서 구역별 잔여석 색상 강도로 빈자리가 있는 구역을 빠르게 파악.
✓ 구현됨 (Case D, LNB 토글)
US-03 구역 드릴다운 탐색 ▼
조감도에서 구역 탭 → 해당 구역 좌석 배치로 줌인. 미니맵으로 현재 위치 확인.
✓ 구현됨 (Case B/D)
US-04 좌석 툴팁 확인 ▼
PC hover / 모바일 1탭으로 좌석 구역·열·번호·가격 확인. 2탭으로 선택.
✓ 구현됨 (전 케이스)
US-05 좌석 선택 및 hold ▼
좌석 선택 시 임시 점유(hold). 재탭으로 취소. 최대 매수 초과 시 토스트 안내.
✓ 구현됨 (전 케이스)
US-06 모바일 핀치 줌 & 패닝 ▼
도면 영역 내 핀치 줌·드래그 패닝. 브라우저 뷰포트 확대 없음. 최대 배율 fit×5.
✓ 구현됨 (모바일)
US-07 공연장 구조물 도면 표시 ▼
무대·토롯코·사운드타워 등 공연장 내 구조물의 위치와 형태를 도면에서 확인.
✓ 구현됨 (Case A/B)
US-08 곡선형 구역 좌석 렌더 ▼
곡선 배치 구역의 좌석이 실제 공연장 배치대로 곡선(arc) 형태로 표시.
✓ 구현됨 (Case C/D)
US-09 좌석 상태 실시간 반영 ▼
다른 사람이 좌석 선택/취소 시 내 도면에 즉시 반영. 이선좌 방지.
✓ 구현됨 (시뮬, LNB 토글)
US-10 구역 교차 좌석 선택 ▼
서로 다른 구역의 좌석을 하나의 예매번호로 함께 선택할 수 있다. FULL 유형에서 기존 구역형의 "동일 구역 내 선택" 제약 해제.
AC-01 FULL 유형에서는 서로 다른 구역의 좌석을 동일 예매번호에 담을 수 있음
⏳ 백엔드 로직 확인 후 확정
US-11 선택 좌석 위치 안내 ▼
선택 좌석에 흰색 ring + 조감도에 구역 인디케이터(흰 원·초록 점) 표시.
✓ 구현됨 (전 케이스, LNB 토글)
US-12 좌석별 유형 표시 ▼
도면에서 특수 좌석(♿ 휠체어석 · ⚠ 시야제한석 · ↺ 자리교환석)의 유형을 아이콘+색상으로 한눈에 확인. 툴팁에서 유형명 추가 표시.
AC-01 특수 좌석에 유형별 아이콘+색상 표시 (이중 부호화)
AC-02 도면 좌하단 유형 범례(legend) 상시 노출
AC-04 유형 미설정 좌석은 일반 좌석 표시 유지
✓ 구현됨 (전 케이스, LNB 토글)
